# better_me

A mobile-first personal OS for tracking daily habits, todos, workouts, body metrics, nutrition, and journaling — with AI-powered insights in a later phase. Built for personal use.

## stack

- **Phoenix** (LiveView + JSON API) + Ecto + PostgreSQL
- **Oban** — background jobs (reminders, streak calculations, digest)
- **Go microservice** — nutrition/macro calculations (Phase 2)
- **pgvector** — vector similarity search for AI insights (Phase 3)
- **Jido** — agent framework for LLM-driven features (Phase 3)

## getting started

```bash
mix setup         # install deps, create DB, run migrations
mix phx.server    # start the server
```

Visit `http://localhost:4000`.

Register at `/users/register`, then log in and visit `/habits`.

## development

```bash
iex -S mix phx.server   # start with interactive shell
mix test                 # run tests (creates and migrates test DB automatically)
mix precommit            # compile + format + test — run before committing
```

Mailbox preview (local email): `http://localhost:4000/dev/mailbox`
LiveDashboard: `http://localhost:4000/dev/dashboard`

## project structure

```
lib/
  better_me/                   # domain logic
    accounts/                  # auth (generated by phx.gen.auth)
    habits/
      schema/                  # Ecto schemas — private to context
        habit.ex
        habit_log.ex
      actions/                 # coordination logic, side effects
        log_habit.ex
        list_with_meta.ex
        current_streak.ex
        recent_logs.ex
        logged_today.ex
      repository.ex            # all Repo calls
      streak.ex                # pure streak calculation
    habits.ex                  # public API — defdelegate only

  better_me_web/               # web layer
    live/
      habits/                  # LiveView pages
    controllers/               # auth controllers
    router.ex
```

Each domain follows a strict three-layer structure: `public API → repository → actions`. See `PRINCIPLES.md` for the full design rules.
